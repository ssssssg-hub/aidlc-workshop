# 테이블오더 서비스 요구사항

## Intent Analysis Summary
- **User Request**: 테이블오더 서비스 신규 구축
- **Request Type**: New Project (Greenfield)
- **Scope**: System-wide (고객 UI + 관리자 UI + 백엔드 서버 + 데이터베이스)
- **Complexity**: Complex (실시간 SSE, JWT 인증, 세션 관리, 이미지 업로드, 다수 기능)

---

## 1. 기술 스택 결정사항

| 영역 | 기술 |
|---|---|
| Backend | Spring Boot (Java/Kotlin) |
| Frontend (고객용) | React (TypeScript) — 모바일 웹 최적화 |
| Frontend (관리자용) | React (TypeScript) — 데스크톱 웹 |
| Database | PostgreSQL |
| API | REST API + SSE (실시간 주문 모니터링) |
| 배포 환경 | Docker Compose 기반 로컬 개발 환경 |
| 인증 | JWT 토큰 기반 |

## 2. 프로젝트 범위

- **매장**: 단일 매장 전용 (MVP)
- **규모**: 소규모 (테이블 20개 이하)
- **UI 구성**: 고객용(모바일 웹)과 관리자용(데스크톱 웹) 별도 앱
- **관리자 계정**: DB seed를 통한 사전 등록 방식
- **이미지 관리**: 서버에 이미지 업로드 기능 포함
- **Security Extension**: 모든 SECURITY 규칙 적용 (blocking constraint)

## 3. 기능 요구사항

### 3.1 고객용 기능 (Customer App — 모바일 웹)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 1회 초기 설정 수행 (매장 식별자, 테이블 번호, 테이블 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 16시간 세션 유지

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면으로 항상 표시
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (페이지 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C04: 주문 생성 및 추가 주문
- 주문 내역 최종 확인 후 주문 확정
- 주문 성공 시: 주문 번호 표시, 장바구니 자동 비우기, 5초 후 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID
- **추가 주문**: 기존 주문 완료 후에도 동일 세션 내에서 새로운 주문 생성 가능 (별도 주문 번호 부여, 동일 세션 ID로 그룹화)

#### FR-C05: AI 메뉴 추천
- 메뉴 화면에 "메뉴 추천" 버튼 제공
- 고객이 인원수, 식사 유형(쉐어/단독) 입력
- 외부 AI/LLM API를 호출하여 인원수·식사유형·매장 메뉴 정보를 기반으로 추천 조합 생성
- AI가 쉐어/단독 여부에 따른 메뉴 구성과 수량을 자동 판단
- 추천 결과 2~3개 조합을 표시, 고객이 선택 후 장바구니에 일괄 추가
- 각 추천 조합에 포함 메뉴명, 수량, 예상 총 금액 표시

#### FR-C06: 주문 내역 조회
- 현재 테이블 세션 주문만 표시
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- 매장 이용 완료 처리된 주문 제외

### 3.2 관리자용 기능 (Admin App — 데스크톱 웹)

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지, 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱, 로그인 시도 제한

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 실시간 주문 목록 업데이트 (2초 이내)
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링

#### FR-A03: 테이블 관리
- 테이블 태블릿 초기 설정 (테이블 번호, 비밀번호, 16시간 세션 생성)
- 주문 삭제 (확인 팝업 → 즉시 삭제 → 총 주문액 재계산)
- **결제 완료 처리**: 테이블별 "결제 완료" 버튼 → 확인 팝업 → 해당 테이블 총 주문 금액 표시 → 결제 완료 확정 시 테이블 세션 종료, 주문 내역 과거 이력 이동, 테이블 주문 목록 및 총 주문액 리셋 (새 고객이 이전 내역 없이 시작 가능)
- 과거 주문 내역 조회 (시간 역순, 날짜 필터링)

#### FR-A04: 메뉴 관리
- 메뉴 CRUD (등록/조회/수정/삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지 (서버 업로드)
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증

## 4. 비기능 요구사항

### NFR-01: 성능
- SSE 기반 실시간 주문 알림 2초 이내 전달
- 소규모 트래픽 (단일 매장, 테이블 20개 이하)

### NFR-02: 보안
- JWT 토큰 기반 인증 (16시간 만료)
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한
- SECURITY-01 ~ SECURITY-15 전체 적용 (Security Extension 활성화)

### NFR-03: 사용성
- 고객용: 터치 친화적 모바일 웹 UI (최소 44x44px 버튼)
- 관리자용: 데스크톱 최적화 대시보드
- 직관적인 카드 기반 레이아웃

### NFR-04: 데이터 관리
- 장바구니: 클라이언트 로컬 저장 (localStorage)
- 주문 이력: PostgreSQL 영구 저장
- 테이블 세션 기반 주문 그룹화

### NFR-05: 배포
- Docker Compose 기반 로컬 개발 환경
- 컨테이너: Backend (Spring Boot), Frontend (고객용 React), Frontend (관리자용 React), PostgreSQL

---

## 5. 제외 범위 (Constraints)

- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 리사이징/최적화, CMS, 광고
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (분석 대시보드, 매출 리포트, 재고, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도, 번역)
