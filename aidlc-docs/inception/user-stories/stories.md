# User Stories

스토리 분류: Feature-Based | 세분화: 개별 동작별 | 수용 기준: 체크리스트

## Epic 1: 테이블 태블릿 자동 로그인 및 세션 관리 (FR-C01)

### US-C01-1: 테이블 초기 설정 입력
**As a** 고객(태블릿 최초 설정 시),
**I want to** 매장 식별자, 테이블 번호, 테이블 비밀번호를 입력할 수 있다,
**So that** 태블릿이 해당 테이블 전용으로 설정된다.

**Acceptance Criteria:**
- [ ] 매장 식별자 입력 필드가 표시된다
- [ ] 테이블 번호 입력 필드가 표시된다
- [ ] 테이블 비밀번호 입력 필드가 표시된다
- [ ] 모든 필드 입력 후 설정 완료 버튼을 누를 수 있다
- [ ] 설정 성공 시 로그인 정보가 로컬에 저장된다
- [ ] 설정 실패 시 에러 메시지가 표시된다

**Persona**: 고객

### US-C01-2: 자동 로그인
**As a** 고객,
**I want to** 태블릿 접속 시 자동으로 로그인된다,
**So that** 별도 로그인 절차 없이 즉시 주문할 수 있다.

**Acceptance Criteria:**
- [ ] 저장된 로그인 정보로 자동 로그인이 수행된다
- [ ] 자동 로그인 성공 시 메뉴 화면이 표시된다
- [ ] 자동 로그인 실패 시 초기 설정 화면으로 이동한다

**Persona**: 고객

## Epic 2: 메뉴 조회 및 탐색 (FR-C02)

### US-C02-1: 카테고리별 메뉴 목록 조회
**As a** 고객,
**I want to** 카테고리별로 분류된 메뉴 목록을 볼 수 있다,
**So that** 원하는 종류의 메뉴를 쉽게 찾을 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴 화면이 기본 화면으로 표시된다
- [ ] 카테고리 탭/버튼이 표시된다
- [ ] 카테고리 선택 시 해당 카테고리의 메뉴만 표시된다
- [ ] 카테고리 간 빠른 이동이 가능하다

**Persona**: 고객

### US-C02-2: 메뉴 상세 정보 확인
**As a** 고객,
**I want to** 메뉴의 상세 정보(메뉴명, 가격, 설명, 이미지)를 확인할 수 있다,
**So that** 주문 전 메뉴에 대해 충분히 파악할 수 있다.

**Acceptance Criteria:**
- [ ] 카드 형태의 메뉴 레이아웃으로 표시된다
- [ ] 각 메뉴 카드에 메뉴명, 가격, 이미지가 표시된다
- [ ] 메뉴 카드 클릭 시 설명을 포함한 상세 정보가 표시된다
- [ ] 터치 친화적 버튼 크기 (최소 44x44px)

**Persona**: 고객

## Epic 3: 장바구니 관리 (FR-C03)

### US-C03-1: 메뉴를 장바구니에 추가
**As a** 고객,
**I want to** 원하는 메뉴를 장바구니에 추가할 수 있다,
**So that** 주문할 메뉴를 모아둘 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴 상세에서 장바구니 추가 버튼이 있다
- [ ] 추가 시 장바구니에 해당 메뉴가 포함된다
- [ ] 이미 있는 메뉴 추가 시 수량이 증가한다

**Persona**: 고객

### US-C03-2: 장바구니 수량 조절
**As a** 고객,
**I want to** 장바구니 내 메뉴의 수량을 증가/감소시킬 수 있다,
**So that** 원하는 수량으로 조정할 수 있다.

**Acceptance Criteria:**
- [ ] 수량 증가(+) 버튼이 동작한다
- [ ] 수량 감소(-) 버튼이 동작한다
- [ ] 수량이 0이 되면 해당 메뉴가 장바구니에서 삭제된다

**Persona**: 고객

### US-C03-3: 장바구니 메뉴 삭제
**As a** 고객,
**I want to** 장바구니에서 특정 메뉴를 삭제할 수 있다,
**So that** 원하지 않는 메뉴를 제거할 수 있다.

**Acceptance Criteria:**
- [ ] 각 메뉴 항목에 삭제 버튼이 있다
- [ ] 삭제 시 해당 메뉴가 장바구니에서 즉시 제거된다

**Persona**: 고객

### US-C03-4: 장바구니 총 금액 실시간 계산
**As a** 고객,
**I want to** 장바구니의 총 금액이 실시간으로 계산되어 표시된다,
**So that** 주문 전 총 비용을 확인할 수 있다.

**Acceptance Criteria:**
- [ ] 메뉴 추가/삭제/수량 변경 시 총 금액이 즉시 갱신된다
- [ ] 각 메뉴별 소계(단가 x 수량)가 표시된다
- [ ] 전체 총 금액이 표시된다

**Persona**: 고객

### US-C03-5: 장바구니 비우기
**As a** 고객,
**I want to** 장바구니를 한 번에 비울 수 있다,
**So that** 처음부터 다시 메뉴를 선택할 수 있다.

**Acceptance Criteria:**
- [ ] 장바구니 비우기 버튼이 있다
- [ ] 비우기 실행 시 모든 메뉴가 제거된다

**Persona**: 고객

### US-C03-6: 장바구니 로컬 저장
**As a** 고객,
**I want to** 페이지를 새로고침해도 장바구니 내용이 유지된다,
**So that** 실수로 새로고침해도 선택한 메뉴를 잃지 않는다.

**Acceptance Criteria:**
- [ ] 장바구니 데이터가 localStorage에 저장된다
- [ ] 페이지 새로고침 시 장바구니가 복원된다
- [ ] 서버 전송은 주문 확정 시에만 수행된다

**Persona**: 고객

## Epic 4: 주문 생성 및 추가 주문 (FR-C04)

### US-C04-1: 주문 내역 최종 확인
**As a** 고객,
**I want to** 주문 확정 전 주문 내역을 최종 확인할 수 있다,
**So that** 실수 없이 정확한 주문을 할 수 있다.

**Acceptance Criteria:**
- [ ] 주문 확정 화면에 메뉴명, 수량, 단가, 총 금액이 표시된다
- [ ] 주문 확정 버튼이 표시된다

**Persona**: 고객

### US-C04-2: 주문 확정
**As a** 고객,
**I want to** 주문 확정 버튼을 눌러 주문을 생성할 수 있다,
**So that** 매장에 주문이 전달된다.

**Acceptance Criteria:**
- [ ] 주문 확정 시 서버에 주문 정보가 전송된다 (매장 ID, 테이블 ID, 메뉴 목록, 총 금액, 세션 ID)
- [ ] 주문 성공 시 주문 번호가 표시된다
- [ ] 주문 성공 시 장바구니가 자동으로 비워진다
- [ ] 주문 성공 후 5초 뒤 메뉴 화면으로 자동 리다이렉트된다

**Persona**: 고객

### US-C04-3: 주문 실패 처리
**As a** 고객,
**I want to** 주문 실패 시 에러 메시지를 확인하고 재시도할 수 있다,
**So that** 주문이 유실되지 않는다.

**Acceptance Criteria:**
- [ ] 주문 실패 시 에러 메시지가 표시된다
- [ ] 장바구니 내용이 유지된다
- [ ] 재시도가 가능하다

**Persona**: 고객

### US-C04-4: 추가 주문
**As a** 고객,
**I want to** 기존 주문 완료 후에도 추가 주문을 할 수 있다,
**So that** 식사 중 추가 메뉴를 주문할 수 있다.

**Acceptance Criteria:**
- [ ] 주문 완료 후 메뉴 화면에서 새로운 메뉴를 장바구니에 추가할 수 있다
- [ ] 추가 주문 시 별도 주문 번호가 부여된다
- [ ] 동일 세션 ID로 그룹화된다

**Persona**: 고객

## Epic 5: AI 메뉴 추천 (FR-C05)

### US-C05-1: 메뉴 추천 요청
**As a** 고객,
**I want to** 메뉴 화면에서 "메뉴 추천" 버튼을 눌러 인원수와 식사 유형을 입력할 수 있다,
**So that** AI가 적절한 메뉴 조합을 추천해준다.

**Acceptance Criteria:**
- [ ] 메뉴 화면에 "메뉴 추천" 버튼이 있다
- [ ] 버튼 클릭 시 인원수 입력 필드가 표시된다
- [ ] 식사 유형(쉐어/단독) 선택 옵션이 표시된다
- [ ] 입력 완료 후 추천 요청 버튼을 누를 수 있다

**Persona**: 고객

### US-C05-2: AI 추천 결과 확인 및 선택
**As a** 고객,
**I want to** AI가 추천한 2~3개 메뉴 조합 중 하나를 선택할 수 있다,
**So that** 고민 없이 적절한 메뉴를 주문할 수 있다.

**Acceptance Criteria:**
- [ ] 추천 결과 2~3개 조합이 표시된다
- [ ] 각 조합에 포함 메뉴명, 수량, 예상 총 금액이 표시된다
- [ ] 조합 선택 시 해당 메뉴들이 장바구니에 일괄 추가된다
- [ ] 추천 로딩 중 로딩 표시가 나타난다

**Persona**: 고객

## Epic 6: 주문 내역 조회 (FR-C06)

### US-C06-1: 현재 세션 주문 내역 조회
**As a** 고객,
**I want to** 현재 테이블 세션의 주문 내역을 확인할 수 있다,
**So that** 지금까지 주문한 내용을 파악할 수 있다.

**Acceptance Criteria:**
- [ ] 현재 세션의 주문만 표시된다 (이전 세션 제외)
- [ ] 주문 시간 순으로 정렬된다
- [ ] 각 주문별 주문 번호, 시각, 메뉴/수량, 금액이 표시된다

**Persona**: 고객

### US-C06-2: 주문 상태 확인
**As a** 고객,
**I want to** 각 주문의 상태(대기중/준비중/완료)를 확인할 수 있다,
**So that** 주문 진행 상황을 알 수 있다.

**Acceptance Criteria:**
- [ ] 각 주문에 상태(대기중/준비중/완료)가 표시된다
- [ ] 매장 이용 완료 처리된 주문은 표시되지 않는다

**Persona**: 고객

## Epic 7: 매장 인증 (FR-A01)

### US-A01-1: 관리자 로그인
**As a** 관리자,
**I want to** 매장 식별자, 사용자명, 비밀번호로 로그인할 수 있다,
**So that** 매장 관리 시스템에 접근할 수 있다.

**Acceptance Criteria:**
- [ ] 매장 식별자, 사용자명, 비밀번호 입력 필드가 표시된다
- [ ] 로그인 성공 시 JWT 토큰이 발급된다
- [ ] 로그인 성공 시 주문 모니터링 대시보드로 이동한다
- [ ] 로그인 실패 시 에러 메시지가 표시된다
- [ ] 로그인 시도 제한이 적용된다

**Persona**: 관리자

### US-A01-2: 관리자 세션 유지
**As a** 관리자,
**I want to** 16시간 동안 세션이 유지된다,
**So that** 영업 시간 동안 반복 로그인 없이 사용할 수 있다.

**Acceptance Criteria:**
- [ ] 브라우저 새로고침 시 세션이 유지된다
- [ ] 16시간 후 자동 로그아웃된다
- [ ] 자동 로그아웃 시 로그인 화면으로 이동한다

**Persona**: 관리자

## Epic 8: 실시간 주문 모니터링 (FR-A02)

### US-A02-1: 테이블별 주문 대시보드 조회
**As a** 관리자,
**I want to** 그리드 레이아웃으로 테이블별 주문 현황을 한눈에 볼 수 있다,
**So that** 전체 매장 상황을 빠르게 파악할 수 있다.

**Acceptance Criteria:**
- [ ] 테이블별 카드 형태의 그리드 레이아웃이 표시된다
- [ ] 각 테이블 카드에 총 주문액이 표시된다
- [ ] 각 테이블 카드에 최신 주문 미리보기가 표시된다
- [ ] 테이블별 필터링이 가능하다

**Persona**: 관리자

### US-A02-2: 실시간 주문 수신
**As a** 관리자,
**I want to** 신규 주문이 실시간으로 대시보드에 표시된다,
**So that** 주문을 즉시 확인하고 처리할 수 있다.

**Acceptance Criteria:**
- [ ] SSE를 통해 신규 주문이 2초 이내에 표시된다
- [ ] 신규 주문 시 시각적 강조(색상 변경, 애니메이션)가 적용된다

**Persona**: 관리자

### US-A02-3: 주문 상세 보기
**As a** 관리자,
**I want to** 주문 카드를 클릭하여 전체 메뉴 목록을 상세히 볼 수 있다,
**So that** 주문 내용을 정확히 확인할 수 있다.

**Acceptance Criteria:**
- [ ] 주문 카드 클릭 시 상세 보기가 표시된다
- [ ] 테이블 번호, 주문 번호, 시각, 메뉴/수량, 총 금액이 표시된다

**Persona**: 관리자

### US-A02-4: 주문 상태 변경
**As a** 관리자,
**I want to** 주문 상태를 대기중/준비중/완료로 변경할 수 있다,
**So that** 주문 처리 진행 상황을 관리할 수 있다.

**Acceptance Criteria:**
- [ ] 주문별 상태 변경 버튼/드롭다운이 있다
- [ ] 상태 변경이 즉시 반영된다
- [ ] 변경된 상태가 대시보드에 업데이트된다

**Persona**: 관리자

## Epic 9: 테이블 관리 (FR-A03)

### US-A03-1: 테이블 태블릿 초기 설정
**As a** 관리자,
**I want to** 테이블 번호와 비밀번호를 설정할 수 있다,
**So that** 해당 테이블의 태블릿이 자동 로그인할 수 있다.

**Acceptance Criteria:**
- [ ] 테이블 번호 입력 필드가 있다
- [ ] 테이블 비밀번호 설정 필드가 있다
- [ ] 설정 완료 시 16시간 세션이 생성된다
- [ ] 성공/실패 피드백이 표시된다

**Persona**: 관리자

### US-A03-2: 주문 삭제
**As a** 관리자,
**I want to** 특정 주문을 삭제할 수 있다,
**So that** 잘못된 주문을 직권으로 수정할 수 있다.

**Acceptance Criteria:**
- [ ] 주문별 삭제 버튼이 있다
- [ ] 삭제 전 확인 팝업이 표시된다
- [ ] 삭제 후 테이블 총 주문액이 재계산된다
- [ ] 성공/실패 피드백이 표시된다

**Persona**: 관리자

### US-A03-3: 결제 완료 처리
**As a** 관리자,
**I want to** 테이블의 결제 완료 버튼을 눌러 주문을 마감하고 테이블을 초기화할 수 있다,
**So that** 새 고객이 이전 내역 없이 시작할 수 있다.

**Acceptance Criteria:**
